# Frontend Stack

Owner: Antoine de Chassey

Last update: 03/09/2025

## Context

---

The client utilizes a proprietary product which manages A&P execution within one of its three existing modules. Originally developed internally a decade ago, this system now requires modernization to align with the company's strategic objectives. The application operates on Node.js, Express, AngularJS, Oracle DB, and Linux. The application today is a monolithic code with approximately 230k lines of code (150k backend / 80k frontend).

## Selection criteria

---

| Criteria | Why this criteria ? |
| --- | --- |
| AI Ready Ecosystem | - Mature technologies with an active ecosystem and extensive documentation. Why? Helps AI provide accurate answers and address frequently encountered issues<br>- Opinionated framework with one good way to do things. Why? Helps AI provide stable answers |
| Migration Distance | - Reduce migration complexity by ensuring origin and target have similarities in syntax and architecture |
| Compliant with Client preferred tech ecosystems & RFP tech requirement | - Must be in Client tech radar<br>- Must enable composable architecture |
| Density of expertise | - Availability of experts to critically challenge and rigorously assess the output generated by AI prompts |

*Note: Server-side rendering (SSR) is not required for this application, as it serves exclusively authenticated users behind login walls and has no search engine optimization requirements.*

## Evaluation

---

| Solution/Criteria | AI Ready Ecosystem | Migration Distance | Compliant with Client preferred tech ecosystems & RFP tech requirement | Density of Expertise |
| --- | --- | --- | --- | --- |
| React.js SPA *+ Vite + ShadcnUI + Tailwind CSS + Tanstack Router + Tanstack Query* | 🟢🟢🟢<br>- Rich and engaged community<br>- Proven large apps in production (Facebook, Netflix, Jira…)<br>- Many tools able to generate code with this stack (Cursor, Copilot, Vercel v0, …)<br>- BUT many ways to do the same thing | 🟠🟠<br>- Extract components from pages<br>- Use React hooks and states instead of $scope<br>- Split Angular JS components in layered domain driven design (app, core, ui) | 🟢🟢🟢🟢<br>- ShadcnUI allows components to be themed and extended easily (100K GitHub stars, used at OpenAI, Adobe, Sonos…)<br>- Based on Tailwind CSS with customizable source code | 🟢🟢🟢<br>- Most popular choice in the tech community |
| Vue.js SPA *+ Vite + PrimeVue + Tailwind CSS + Tanstack Query* | 🟢🟢<br>- Fewer proven large apps<br>- Good TypeScript support, 2nd largest frontend framework (Louis Vuitton, Xiaomi) | 🟠🟠<br>- Extract components from pages<br>- Use Vue composition API and refs instead of $scope<br>- Split Angular JS components in layered domain driven design (app, core, ui) | 🟢🟢<br>- PrimeVue has prebuilt components but is less extendable than Shadcn (13K GitHub stars) | 🟢🟢<br>- Fast-growing popularity in the tech community |
| Angular SPA *+ Material UI + Tailwind CSS + Tanstack Query* | 🟢🟢<br>- ~2x smaller community than React but strongly maintained by Google<br>- Claude and ChatGPT are less trained on Angular compared to React<br>- Angular has recently adopted signals to replace RxJS deprecating many code examples<br>- BUT less historical variability in architecture choices | 🟠🟠<br>- Extract components from pages<br>- Use Angular services and signals instead of $scope<br>- Significant architectural changes with Angular 2+ (components, services, dependency injection)<br>- Split Angular JS components into complete rewrite of Angular components | 🟢<br>- Material UI is more opinionated (25K GitHub stars, used in Google products) | 🟠<br>- Slowly losing market-share in the tech community |

## Recommendation

---

**React.js + Vite + ShadcnUI + Tailwind CSS + Tanstack Router/Query** is a modern and strong community backed tech stack. Extensive documentation and a broad ecosystem is a great advantage to help AI build great UI components.
